Blogy API Docs

The official REST API documentation for **Blogy**. Built for speed and reliability using Django & DRF.

Explore in Postman →

API Endpoints

  • GET /blog_list/
    Retrieve a list of all public blog posts
  • POST /register_user/
    Create a new user account (Public)
  • POST /create_blog/
    Protected: Create a new post
  • PUT /update_blog/<int:pk>/
    Protected: Update a specific post
  • DELETE /delete_blog/<int:pk>/
    Protected: Delete a specific post
  • PUT /update_user/
    Protected: Modify user profile

JWT Authentication Flow

Protected endpoints require an access token from the **DRF Simple JWT** system. Follow these steps to authenticate:

  1. Create an account using the POST /register_user/ endpoint.
  2. Acquire your tokens by sending a POST request to the token acquisition endpoint: /token/ (aliased as token_obtain_pair).
  3. For all subsequent protected requests, include the token in your HTTP headers: Authorization: Bearer <your_token>.
  4. (Optional) Refresh your token using POST /token_refresh/ and your refresh token.

Example: Obtaining a token with curl 

# Use your production domain or local host for the base URL
curl -X POST \
  -H "Content-Type: application/json" \
  -d '{"username":"your_username","password":"your_password"}' \
  https://api.blogy.com/token/

# Example Local Host:
curl -X POST \
  -H "Content-Type: application/json" \
  -d '{"username":"your_username","password":"your_password"}' \
  http://127.0.0.1:8000/token/
